# 08 — 抽取功能

**What to build:** 实现首页核心抽取交互：前置校验、触发抽取、卡片翻转动画（手动逐张）、单张重抽、全部翻开后用户手动「确认」写入历史，以及首页今日摘要展示。

**Blocked by:** 02 — Draw Engine、05 — 菜品池管理、07 — 抽取方案管理、11 — 群组数据模型

**Status:** done

- [x] 首页展示「开始抽取」按钮；读取 `activeDrawConfigGroupId` 解析当前生效方案的 `entries`，使用 `validateDrawConfig` 实时校验菜品池，不满足时按钮禁用并展示具体原因（「${分类名}可用菜品不足，需至少 ${count} 道」）
- [x] 首页 idle 区域顶部展示方案快捷切换入口：显示当前方案名 + `›` 箭头，点击弹出 ActionSheet 从 `drawConfigGroups` 列表中切换，即刻写入云端并重新校验
- [x] 方案切换不展示 loading 态，避免页面闪烁
- [x] 点击按钮调用 `drawDishes(pool, activeGroup.entries)` 生成结果，以背面朝上的卡片阵列展示（数量 = activeGroup.entries 求和）；卡片背面为绿色渐变 + `?` + 分类名
- [x] 每张卡片可独立点击触发 CSS 3D 翻转动画（`rotateY(180deg)` + `backface-visibility`），翻转后正面（浅绿底色 + 正绿边框）显示菜品名 + 第一张图片（无图则纯文字）
- [x] 卡片区域上方提供「一键翻牌」按钮（outlined 样式），点击后按顺序逐张翻转（400ms 间隔），翻牌期间按钮禁用，开始新抽取/返回/确认后重置
- [x] 已翻转的卡片右下角显示「换一道」按钮；点击后卡片先翻回背面，再在同分类内随机替换一道（排除当前已抽取的所有同分类菜品），再手动翻转揭晓
- [x] 重抽不跨分类；换一道排除该分类内所有已分配卡片（含未翻开），非仅已翻开的
- [x] 底部操作栏（fixed）：左侧「返回」纯绿文字，右侧「重抽」+「就吃这个」；`justify-content: space-between`
- [x]「返回」：弹窗确认「返回后当前抽取结果将不会保留」，无论是否有卡片已翻开
- [x]「重抽」：仅在至少一张卡片翻开后启用（否则置灰），点击后按当前方案重新抽取全部卡片
- [x] 所有卡片翻开后「就吃这个」按钮激活（翻开前按钮置灰不可点）
- [x] 点击「确认」将本次结果写入 `draw_history`（status: 'active', `drawerId`: 当前用户 openid），写入后检查并将 7 天滚动窗口外的记录软删除（status: 'archived'）
- [x] 历史写入冗余字段：`categoryName`、`dishName`（防分类/菜品被删后显示异常）；时间字段为 `confirmedAt`
- [x] 确认后页面回到待机状态，支持再次发起新一轮抽取
- [x] 首页常态展示今日最新一条已确认结果 + 今日已确认条数摘要（如「今天已确认 2 餐」）
- [x] 点击首页摘要区域跳转至历史页
- [x] 今日无已确认记录时展示引导占位（如「今天还没决定吃什么，点击抽取」）
- [x] `onShow` 在同群组内也重新加载 config（修复从子页面返回后方案列表未刷新、历史记录未刷新的问题；index 页在 drawing 态跳过以免打断）
