# 03 — 插件管理页

**What to build:** 用户打开插件管理页后，看到所有可用插件列表——每个插件展示名称、描述、解锁状态及进度、启用/禁用开关。用户可手动触发解锁、切换开关，操作结果即时反馈。

**Blocked by:** 01 — plugin-manage 云函数

**Status:** done

- [x] 插件管理页进入时调用 `plugin-manage` 的 `list` 获取全量数据（插件定义 + 用户状态 + 评估进度）
- [x] 列表展示每个插件的名称、描述
- [x] 未解锁插件展示解锁进度（如"累计创建 3/5 道菜品"）和解锁按钮
- [x] 已解锁插件展示"已解锁"标记和启用/禁用开关
- [x] 点击解锁按钮 → 调 `unlock` → 达标则即时更新为已解锁并展示开关 → 不达标则展示失败提示和当前进度
- [x] 切换开关 → 调 `toggle` → 即时生效
- [x] 解锁或切换操作失败时有错误提示
- [x] 管理页进入时独立调用 `list`（不依赖 `plugin-store` 缓存），确保显示实时数据
- [x] 页面路径在 `app.json` 中声明
