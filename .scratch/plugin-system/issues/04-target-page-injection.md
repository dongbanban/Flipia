# 04 — 目标页面 WXML 注入 + 示例插件

**What to build:** 一个具体可运行的示例——在菜品详情页看到已启用插件的组件渲染，禁用后消失。端到端验证插件管理页开关 → 目标页渲染生效的完整链路。

**Blocked by:** 02 — plugin-store + 启动集成；03 — 插件管理页

**Status:** ready-for-agent

- [ ] 创建一个示例插件组件（占位内容即可），验证注入机制
- [ ] 在 `plugin-manage` 云函数的 `pluginDefs` 中注册该示例插件（含 `assess` 函数，设定一个简单的解锁条件）
- [ ] 在目标页面（如菜品详情页）WXML 中加入该插件的组件标签，`wx:if` 绑定 `plugin-store.isEnabled()`
- [ ] 插件启用后组件在目标页可见；禁用后消失
- [ ] 全程无需修改 `plugin-store` 或管理页——仅新增插件定义、组件、页面 WXML 标签三处
- [ ] 属于专属页面的插件：在 `app.json` 中声明页面路径，管理页中提供入口
