# Flipia

个人/家庭日常菜单随机抽取工具。以群组为数据归属单元，在群组内维护菜品池、按分类组织菜品，通过抽取方案随机决定每餐吃什么。

## Language

**群组 (Group) / 厨房**：
数据归属的唯一单元。每个用户至少拥有一个群组，所有菜品、分类、方案、抽取历史均挂在群组下。不存在独立于群组的"个人数据"——单用户场景即成员数为 1 的群组。一个用户可以属于多个群组，在使用时切换当前活跃群组。**厨房**是群组面向用户的展示文案——miniprogram 目录下编写代码时使用 `group`，展示文案均使用"厨房"。
_Avoid_: 家庭、团队、仓库、空间

**菜品 (Dish)**：
群组内维护的每道菜，有名称、所属分类、图片、启用/禁用状态，以及创建者标识（用于展示，非权限控制）。群组内所有成员对菜品有完全操作权限（增删改禁用）。
_Avoid_: 食品、菜式

**创建者 (Dish Creator)**：
菜品在群组内的添加人标识，仅用于展示（如"红烧排骨 大厨：张三"），不赋予特殊权限。
_Avoid_: 作者、添加人

**分类 (Category)**：
菜品归属的类别（如荤菜、素菜、主食、汤），用于组织菜品池和作为抽取方案的 entry slot。新建群组时自动预置四个默认分类作为模板，所有分类均可自由增删改名——不存在不可变的系统分类。
_Avoid_: 类别、分组、标签

**菜品池 (Dish Pool)**：
某个群组内某个分类下的全部菜品集合，是抽取的数据来源。

**导入 (Import)**：
将菜品从用户已有的某个群组复制到另一个群组的操作。导入后两份数据独立演进（复制语义），互不影响。
_Avoid_: 迁移、合并、同步

**抽取方案 (Draw Config Group)**：
一套抽签规则，定义参与抽取的分类及每类抽取数量（1-5 道）。用户可在每个群组内保存多套方案（最多 10 套），不同场景切换不同方案。
_Avoid_: 配置组、抽签模板、抽取配置

**生效方案 (Active Draw Config Group)**：
当前群组内标记为"使用中"的抽取方案。首页抽取按钮始终读取生效方案的规则。

**抽取 (Draw)**：
按生效方案的规则从菜品池中随机选取菜品的动作，使用 Fisher-Yates 洗牌。

**抽取结果 (Draw Result)**：
一次抽取的输出，由每分类的菜品列表组成。用户手动确认后写入历史，记录抽取人。

**历史记录 (Draw History)**：
已确认的抽取结果，保留 7 天滚动窗口，超期记录软删除（archived）。历史记录按天分组，每条标记抽取人。数据归属于群组，成员退出后不带走。

**解散 (Dissolve)**：
群组生命周期终点。当群组仅剩一人且该成员执行退出操作时，群组及其所有数据被永久销毁。也可由创建者主动解散。不可逆。
_Avoid_: 删除群组、清空

**群主 (Group Owner)**：
群组的创建者。拥有踢人和解散群组的独占权限。其他操作（邀请、退出、增删改数据、修改群组名）对所有成员开放。
_Avoid_: 管理员、群管理员
