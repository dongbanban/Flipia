# ADR-0007: 引入用户级数据层——插件状态归属用户而非群组

**Status:** accepted

## Context

ADR-0002 确立了"一切皆群组"原则：不存在独立于群组的数据。插件体系建设中需要决定插件解锁与开关状态的数据归属粒度。

## Decision

**插件状态归属于用户级别。** 每个用户拥有独立的插件解锁进度和开关偏好，不随群组切换而变化。

- 解锁条件评估基于用户跨群组的累计行为（如总创建菜品数、总抽取天数）
- 启用/禁用开关是用户级偏好，跨设备云同步
- 插件能力的注入目标可以是群组维度（如"在菜品详情页插入营养信息"），但插件是否激活由用户级状态决定

## Considered Options

- **群组级插件状态** — 每个群组独立解锁和开关。用户在不同群组需要重复解锁同一插件，切换群组时体验割裂。拒绝。
- **用户-群组交叉状态** — 在群组 A 里解锁不等同于在群组 B 解锁。复杂度翻倍且无明确使用场景。拒绝。

## Consequences

- 需要新建用户级数据存储（当前数据库以群组为唯一归属单元）
- 插件状态需要云函数端评估和持久化（防篡改、跨设备同步）
- CONTEXT.md 新增 User、Plugin、Unlock Condition、Plugin State、Plugin Registry、Extension Point 术语
- ADR-0002 的"不存在独立于群组的数据"约束被放宽：插件状态是该原则的现有唯一例外
